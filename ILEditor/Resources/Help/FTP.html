Something not working?

Are you seeing this error message?

This may be because the Windows Firewall is blocking either:

  • ftp.exe (the Windows FTP client)
  • Port 20
  • Port 21
  • You're using Parallels (see bottom)

Follow these steps:

  1. Open the Windows Firewall
  2. Click 'Allow an app or features through Windows Firewall' in the top left
  3. Find 'File Transfer Program' in the list shown - there may be multiple.
  4. Make sure all the 'File Transfer Program' items are ticked - as well as the public and private columns.
  5. If you didn't see them, select 'Allow another app...'
  6. You will need to add two 'ftp.exe' programs to the list:
    1. C:\Windows\SysWOW64\ftp.exe
    2. C:\Windows\System32\ftp.exe
    Please take into account which drive you installed Windows on.
  7. Retest what wasn't working.

Still not working? Check out these guides:

  • Add ftp.exe to the firewall (video) - after you have done this, test again as you may not need to unblock the ports if this works.
  • How to Allow FTP Traffic Through Windows 8, 8.1 and Windows 7 Firewall (article) - also works for Windows 10
  • IBM i QuickTip: When You Can’t FTP from a Windows Desktop to an IBM i (article)

Using ILEditor within Parallels (Mac)

If you're having the issue where you can't get a number of features working while using ILEditor in a Parallels VM, you need to change the default network adapter to be bridged.
